# Source Verification

Catalog records should be traceable to a publisher-controlled source whenever possible.

## Verification checklist

1. Open the original publisher page.
2. Confirm that the dataset or catalog entry still exists.
3. Confirm the displayed publisher name.
4. Confirm the dataset title in both available languages.
5. Record the declared license exactly as shown.
6. Confirm the source URL is stable enough for citation.
7. Record the verification date in ISO format.
8. Note whether access requires registration, approval, or payment.
9. Avoid copying source data or long descriptions into this repository.

## Evidence quality

Preferred evidence order:

1. publisher-controlled dataset page;
2. publisher-controlled documentation or license page;
3. official government or institutional catalog;
4. trusted archive of an official page;
5. secondary directory used only as a discovery aid.

## Unavailable sources

When a source becomes unavailable:

- do not silently replace it with an unrelated mirror;
- record the last successful verification date;
- add a status note;
- look for an official replacement page;
- preserve the correction in Git history.

## License uncertainty

A license label is metadata, not a legal conclusion. When the source terms are unclear, use a review status and avoid claiming that commercial reuse is allowed.

## Review record

A verification commit should identify the affected record IDs, source pages checked, changed fields, reviewer, and verification date.
